

## Scroll-driven видео в Hero (только десктоп)

### Что делаем
При скролле вниз на десктопе видео в фоне Hero плавно проигрывается синхронно с прокруткой. Мобильная версия остаётся без изменений (статичная картинка).

### Шаги

1. **Сохранить видео** — конвертировать `.mov` в `.mp4` (для браузерной совместимости) и сохранить в `public/hero-video.mp4`

2. **Создать хук `useScrollVideo`** (`src/hooks/useScrollVideo.ts`)
   - Принимает `ref` на `<video>` и `ref` на контейнер секции
   - Слушает `scroll`, вычисляет прогресс (0–1) по позиции контейнера
   - Устанавливает `video.currentTime = progress * duration`
   - Использует `requestAnimationFrame` для плавности

3. **Обновить `HeroSection.tsx`** (только десктопная ветка)
   - Секция получает `height: 300vh` — это даёт пространство для скролла
   - Внутри — `sticky`-контейнер `height: 100vh` с видео в фоне
   - `<video>` заменяет фоновое изображение: `muted`, `playsInline`, `preload="auto"`, без `autoPlay` и `controls`
   - Текст и кнопки остаются поверх как overlay
   - Если видео не загрузилось — fallback на текущую картинку `heroBg`

4. **Мобильная версия** — без изменений, остаётся как есть

### Технические детали
- Конвертация `.mov → .mp4` через `ffmpeg` (уже доступен в sandbox)
- Видео в `public/` (не в `src/assets/`) — чтобы избежать проблем с бандлингом больших файлов
- `position: sticky` + высокий контейнер — стандартный паттерн для scroll-driven анимаций
